Telnets into GNS3 routers and injects the following misconfigurations (no write memory):
  • R1 — Interface FastEthernet1/0 shutdown
  • R2 — EIGRP passive interface on Fa1/0; EIGRP stub mode enabled
  • R3 — EIGRP non-default K-values (weights 0 2 0 1 0 0); Timer mismatch on Fa0/0 (hello=1s, hold=3s)
  • R4 — OSPF timer mismatch on Serial0/0 (hello=30s, dead=120s)
  • R5 — OSPF area 1 stub; Network 192.168.10.0/24 in wrong area (area 1 instead of area 0); Duplicate RID 1.1.1.1
  • R6 — OSPF passive interface on Fa0/0; Duplicate RID 1.1.1.1 (same as R5)
Launches a TCP SYN flood attack demonstration:
  1. SSHs into Kali VM → runs tcp_flood.py with sudo (generates attack traffic)
  2. SSHs into Host PC → starts CombinedIDS.py for 15 seconds
  3. Naive Bayes model detects the flood → IDS alert sent to this GUI
  4. Both scripts terminated automatically after 15 seconds
After detection, click Run Diagnostics to apply the fix (shuts R6 Fa0/0).
Launches an OSPF Hello flood attack demonstration:
  1. SSHs into Kali VM → runs ospf_attack_v2.py with sudo (floods OSPF hellos)
  2. SSHs into Host PC → starts CombinedIDS.py for 15 seconds
  3. Random Forest model detects the OSPF flood → IDS alert sent to this GUI
  4. Both scripts terminated automatically after 15 seconds
After detection, click Run Diagnostics to apply the fix (shuts R6 Fa0/0).
